## 🚀 Project Overview

The modern recruitment process is heavily guarded by Applicant Tracking Systems (ATS). Highly qualified candidates face rejection simply due to poor formatting or missing keywords.
**Rizzume AI** addresses this challenge by providing a **predictive, analytical, and scalable ATS bypassing system** that supports:

* Real-time ATS compatibility scoring
* Identification of critical missing skills
* Data-driven career preparation strategies

Unlike standard keyword matchers, Rizzume AI is **context-aware**, deeply analytical, and production-ready via microservices.

## 🧩 System Architecture

Rizzume AI follows a decoupled **three-tier client–server architecture**:

### 1️⃣ Machine Learning Layer (Python / FastAPI)
* Secure PDF bytes processing via `PyPDF2`
* Text cleaning & preprocessing
* TF-IDF mathematical text vectorization
* High-speed asynchronous endpoints

### 2️⃣ Orchestration Layer (Node.js / Express)
* REST API orchestration
* File buffering using `Multer`
* Generative AI (Groq/Gemini) integration
* Cross-service data streaming to Python engine

### 3️⃣ Frontend Layer (React.js)
* Highly responsive UI built with Tailwind CSS
* Secure file drag-and-drop zones
* Interactive dashboards for ATS Score, Roast, and Roadmap

## 📊 Core ML Logic: How the ATS Score Works

* **Weight Matters:** Using TF-IDF, it understands that the word "Kubernetes" in a DevOps JD carries significantly more mathematical weight than the word "Teamwork".
* **Vector Comparison:** The system creates vectors for both the Resume and the Job Description, calculating the cosine similarity to generate a highly accurate percentage.

---

## 🚀 Deployment & DevOps Engineering

**Overcoming Cloud Limitations:**
Both backend services are hosted on Render's free tier. To prevent `503 Timeout` errors during cross-service communication:
* Engineered custom `/health` routes on both Python and Node servers.
* Implemented specific schedule-based Cron-jobs to ping the central API during daytime hours, ensuring high availability without exhausting the monthly cloud limit.
